2020年5月12日 晚上
598 字 6 分钟

1、起因

闲来无事酷安，偶然间看到有人在刷语雀的推广码，就点进去看了一下，发现 UI 和功能都很能打动我，遂注册登录下载，然后去知乎搜了一下看看大家都怎么看语雀，还是好评居多。又碰巧瞥到了一个 hexo 与语雀结合的案例，并且用 Travis CI 进行自动更新，真是太香了啊，开整。

2、过程

从八点钟开始折腾了一晚上，先装了 Ruby，再装 Msys2，结果到最后还是装不上 Travis，一直到后半夜也没搞好。到后来又看到了 GitHub Actions 这个东西，和也能够实现持续集成，结果也还是搞不好，文件、SSH、Token 自我检查感觉全都没问题，结果却一直不成功，好吧，一直折腾到 4 点钟，放弃睡觉。

3、转折

后来放弃了持续集成这个念头，决定就用 yuque-hexo 加上自己勤劳的双手手动更新吧，然后看上了一款 hexo 的主题：halo，接着开整，结果啊主题下载后，各种文件也配置了，yml 写的也没啥问题，结果一直报错，网上能找到的解决方案寥寥无几也都没有效果，从 10 点一直折腾到下午 2 点，放弃。换一款主题，结果三分钟搞定，虽然效果没有那么炫酷，但是简单、整洁：fluid。找了一下好看的图片，把主题原来的浅色配色改成了深色，总算是告一段落了。
成果就是这样了。

4、总结

赏心悦目的设计和酷炫的效果确实诱人，显示现在回想起来为了这写耗费了一整天的时间，是不是值得呢？如果自己的前端知识更丰富一点是不是就不会费这么大的力气了呢？将这些时间用来学习，用原来简洁的 next 主题也很好呀。
喜欢折腾不是坏事，但是今天的做法我觉得自己有些本末倒置了，博客最大的作用应该是记录自己的学习和生活轨迹，分享自己的知识和经验，内容才是最重要的，还好我及时转变了思想，现在的这款主题用着也是很香的，嘻嘻。
